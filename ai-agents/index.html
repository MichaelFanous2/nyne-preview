# Nyne as a tool for a Claude agent (Anthropic SDK)
import anthropic, requests

client = anthropic.Anthropic()

tools = [{
    "name": "nyne_person_enrich",
    "description": "Verified context on a person (demographics, "
        "residence, wealth, affinities, life events) from an email.",
    "input_schema": {"type": "object",
        "properties": {"email": {"type": "string"}},
        "required": ["email"]},
}]

def nyne_person_enrich(email):
    r = requests.post("https://api.nyne.ai/v1/person/enrich",
        headers={"x-api-key": NYNE_API_KEY}, json={"email": email})
    return r.json()  # typed person, confidence-scored

response = client.messages.create(
    model="claude-opus-4-8",
    max_tokens=16000,
    tools=tools,
    messages=[{"role": "user",
               "content": "Qualify this lead: c.hutchins@example.com"}],
)

for block in response.content:
    if block.type == "tool_use":
        context = nyne_person_enrich(**block.input)

FAQ

Agents & Nyne.

How do AI agents use Nyne?

Expose Nyne's enrich and search endpoints as tools. When the agent needs context about a person, it calls the tool, receives typed JSON with confidence scores, and acts on it: qualifying a lead, routing a workflow, or personalizing outreach.

Which agent frameworks does Nyne work with?

Any framework that supports tool or function calling: the Anthropic Claude API, OpenAI function calling, LangChain, and custom agent loops. Nyne is a standard HTTPS JSON API, and webhooks push live signals into agent workflows.

Why do agents need a people data API?

Agents act on whatever context they have. Without verified people data they guess from sparse CRM fields. Nyne gives them the resolved person, household and company with provenance and confidence scores, so decisions are grounded in verified facts rather than inference.
